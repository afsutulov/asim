# Что удалось установить по архивам

## 1. Что читает исходный `cron`

Из `config.json` проект `cron` использует поля:

- `sentinel`
- `logs`
- `results`
- `results_path`

Лог пишет в:

- `<logs>/cron.log`

## 2. Как `cron` строит `cash.json`

Для каждого каталога вида:

```text
<sentinel>/<year>/<SAFE>.SAFE
```

он делает следующее:

1. Парсит имя SAFE-пакета.
2. Ищет путь:

```text
<SAFE>.SAFE/GRANULE/<granule>/IMG_DATA
```

3. Читает облачность из `MTD_MSIL1C.xml`.
4. Берёт первый найденный JP2 в `IMG_DATA`, чтобы вычислить географический envelope.
5. Сохраняет результат в `<sentinel>/<year>/cash.json`.

## 3. Подтверждённая структура `cash.json`

По коду `cron` структура такая:

```json
{
  "version": 7,
  "preprocess": "sentinel",
  "generated": "2026-03-19T10:00:00Z",
  "year": 2025,
  "tiles": [
    {
      "date": "2025-06-01",
      "captured_at": "2025-06-01T08:15:31Z",
      "scene_id": "T40VEP|2025-06-01T08:15:31Z",
      "safe_name": "S2A_MSIL1C_20250601T081531_N0511_R121_T40VEP_20250601T102233.SAFE",
      "img_data_path": "/.../GRANULE/.../IMG_DATA",
      "cloud": 0,
      "envelope": [54.1, 57.2, 55.3, 58.1],
      "tile_id": "T40VEP"
    }
  ]
}
```

Ключевые поля для новой задачи:

- `captured_at`
- `img_data_path`
- `cloud`
- `tile_id`
- `safe_name`

## 4. Что не удалось подтвердить напрямую

В архиве `data.tar.gz` **не было годовых папок Sentinel** (`2023`, `2024`, `2025`, ...), а в архиве `cron.tar.gz` **не было примеров `cash.json`**, только код индексации.

Поэтому в новом проекте использованы:

- **точно подтверждённый формат** `cash.json` из исходников `cron`;
- **стандартная структура Sentinel-2 L1C** для поиска RGB-каналов `B04/B03/B02` внутри `IMG_DATA`.

## 5. Что делает новый проект

Новый проект:

- читает `config.json`;
- берёт `sentinel`, `results_path`, `logs`;
- читает `cash.json` по всем годам диапазона дат;
- выбирает по каждому tile Пермского края самый поздний снимок с `cloud == 0`;
- собирает RGB из `B04/B03/B02`;
- нормализует яркость/контраст тайлов к общим статистикам;
- строит итоговую мозаику;
- сохраняет её как JP2 с геопривязкой;
- ведёт подробный лог в `mosaic.log`.
